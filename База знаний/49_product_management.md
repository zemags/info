### продуктовая идея может вызникнуть из:
- анализ рынка\тренды\аудитории\конкуренты
- если есть изменения в "привычном поведении", смена\не смена метрик
- появление\изменение технологий
- из похожих твоей области деятельности
- costumer development - качественные исследования - тестирование на востребованность продкута с помощью потенциальных потребителей

#### анализ рынка - оценка рисков и возможностей
- объем рынка - реальные продажи в отрезок времени (позиция конторы на рынке, динамика\конкуренция)
- емкость рынка - максимальные продажи в теч времени которые можно достигнут
- тенденции рынка
	- pestel анализ (политические, экономические, социальные, технологическе, экологические, юридические тренды влияющие на отрасль)
- конкуренты
	- пять сил Портера (угроза появления новых игроков\заменителей, власть поставщиков\покупателей, конкуренция)
- текущие решения удовлетворяют потребность?

#### продуктовая гипотеза - идею нужно проверить, экономить ресурсы, быстрый результат от рынка
- гипотезы:
	- потребности, проблемы, ценности, роста,

#### формулировка гипотез
1. описание изменения
2. описание эксперимента, метрики
3. условие или значение метрики

#### проверка гипотезы
- фокус группа, глубинные интервью
- опросы, аналитика данных
- а\б тестирование (пример: половина юзеров переход по красной кнопке, другая половина по зеленой, и в теч недели смотрит метрики)
- можно через краудфандинг, насколько идея интересна

#### постановка целей
- техника smart - гипотиза не бредова, влияет на метрику, ведет к результату
	- **s** - specific - цель
	- **m** - measurable - измеримые метрики
	- **a** - attainable - достижимые идеи(трезво смотреть)
	- **r** - relevant - значимые - точно ли нужна гипотеза
	- **t** - timebound - четкие сроки временные

#### циклы hadi
- **h** - hypothesis - гипотеза
- **a** - actions - дейсвтия
- **d** - data - данные
- **i** - insights - выводы

#### целевая аудитория - общие признаки для группы людей
- нужно знать аудиторию иначе провал

#### сегментирование аудитории - по признакам разделить
- критерии:
	- социально-демографические (пол, возраст, доход ..)
	- стиль жизни, соц положение, личные качества
	- локация\ география (страна ..)
	- поведение (оплата картой в инете ..)
- как определить аудиторию
	- методика **5W**
		- **what** - какой продукт, **who**, **why**, **when**, **where**
- где использовать:
	- история пользователей, аналитика метрика, рекламная активность, сценарии использования

#### качественные исследования
- интерью с экспертами (кто реально щарит за боль пользователя)
- глубинные интервью (custDev)
- фокус группа
- моделирования для пользователя ситуации

#### количественные исследования
- тесты (а\б, коридорные исследования), опросы

#### источники инфы
- опен сорс, форумы, соц сети, чаты, группы, отзывы

#### jobs to be done - теория о поведении пользователей
- продукт должен выполнять конкретную функию, юзер ищет продукт для конкретной задачи
- этот продукт "мечта" юзера
- мотивация юзера, вектор развития, почему продукт в прошлом покупали, проблемы юзеров

### costumer development - быстрый фидбек от юзера от пользования продуктом
- с помощью глубинных интеревью с юзером
- сократить расходы на все
- появление новых идей
- возможный отказ от продукта в случае отсуствия интереса аудитории
- этапы:
	1. обнаружение клиентов для продукта - **search** - здсь custdev
	2. подтверждение клиентов - **search**
	3. создать клиентов - **execution** -запуск продукта
	4. построение конторы - **execution** - запуск

#### product market fit - любовь юзера к продукту
- переход от **search** к **execution**

#### виды интервью
- сценарные - путь юзера
- проблемные - боль юзера
	- 5 шагов
		1. выбор сегмента - аудитория
		2. выбор гипотезы, проверяем
		3. подготовка сценария для интервью
		4. провести поиск юзеров интервью
		5. анализ выходных данных - нашли боль\инсайты, сегментируем
	- слушаем юзера, непредвзято всё, не нужно толкать идея, важно понять боль юзера
- решенческие - подтвереждение ценности продукта

#### вопросы для интервью
- открытые вопросы четкие, вопросы для получения развернутого ответа, нужна инфа
- как прошел прошлый опыт использования продукта, про будущее продукта неинтересно
- оперировать фактами

#### ценностное предложение - прозрачные плюсы продукта в отличие от конкурента (допустим на главное странице надпись, которая цепляет)
- четко сформулировано, уникально для продукта, и выгода какая для юзера
	- какой продукт, потребитель, какую боль решает, чем лучше других, какие выгоды

#### разработка ценностного предложения - глубокое понимание ценности и развития продукта
- шаблон остервальда
	- портрет юзера
		- какие задачи решает продукт
		- проблемы юзера которые могут возникнуть
		- выгода желаемая ожидаемая
	- карта ценности

#### юнит экономика продукта - оценка эффективности продукта с помощью мат. расчетов
- юнит это клиент, договор, ноый юзер и тд (например покупатель в магазе)
- задачи
	- запуск продукта А или В
	- сколько юзеров привлечь
	- наскольки эффективны продвижения продукта
	- поиск слабых мест в продукте
	- когда будет прибыль

#### метрики юнит экономики
- для одного юнита = расход / доход
	- из чего состоит расход (издержки постоянные(аренда офиса), переменные(заказ товара)) и доход

#### юнит
- customer acquisition cost (cac) - стоимость привлечения клиента в продукт
- lifetime value (ltv) - за все время юзера в продукта сумма дохода
- cac > ltv - плохо
- ltv > cac - хорошо
- average price unit (arpu) - средний чек (avp) = доход/ кол-во заказов
- прибыль = (avp - переменные затраты) / avp

### формирование пользовательских сценариев
- для проектирования интерфейса продукта
- обратить внимание на выжные моменты
- понимание развития продукта, этапы
- понять аудиторию и сделать лучще работу юзера с продуктом
- примерить роль юзера на себе

#### создание юзер стори - короткое описание что сервис должен предоставить юзеру
- создать персону (например администратор) - роль важна
- действия для сценария
	- в нем цель, среда, мотивация, короткие и понятные действия
- оставить только необходимое, ценность
- как писать юзер стори - хронологический порядок с приоритетом и дедлайнами
	- метод invest
		- **i** - independent не зависимо от других сторей
		- **n** - negotiable обсуждение с командой
		- **v** - valuable ценность
		- **e** - estimate трудозатраты
		- **s** - small реализация за одну итерацию
		- **t** - testable - критерии выполнения и тестирования

#### карта клиентского пути - как юзер взаимодействут с продуктом
- разработка и прогноз всех шагов юзера
- поиск слабых мест и сильных мест
- оптимизация пути для шагов юзера

#### прототипирование продукта
- **mvp** - minimal viable product - минимум для удовлетворения хотелок юзера
- цели
	- проверка гипотезы, мало кода, мало затрат, быстрый фидбек от первых юзеров, готовы они платить
	- приоритет фич, узнать о аудитории, данные о потенциальных сегментах - куда развиваться
	- коррекция направления развития продукта

#### виды mvp
- краудфандиг, одностраничник с кнопкой "купить",
- видеопрезентация, концентрация на одной фичи (типа поиск в интернете)
- макет от руки нарисован

#### метрики в mvp
- бизнес метрики
	- ltv
	- cac
	- средний чек юзера
- интерфейсные и пользовательские
	- кнопки, формы, частота использования, взаимодейсвтия с юзером
	- проблемы юзер стори, отток и возврат клиентов, чем фича А хуже в сравнении с В
- качество
	- отказоустойчивость, скорость

#### запуск mvp
- анонс
- анализ результатов
	- генерация\подтверждение\опровержение гипотез, баг фикс, сбор данных, новые данные об аудитории
